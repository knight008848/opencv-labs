## 概念 A：透视变换

### 一句话解释
透视变换用 4 对点将倾斜视角的平面物体"拉正"为俯视图——模拟"近大远小"的反向操作。

### 生活类比
你站在马路中间低头看地面的斑马线——近处的线条很宽，远处的线条很窄甚至汇聚成一个点。透视变换就是把这个"歪斜视角"拍的照片变成"正上方俯拍"的效果——像 Google Earth 卫星图那样，所有的线都恢复了真实的比例。

### 技术解释
透视变换由 4 对对应点确定一个 3×3 的单应矩阵。这个矩阵描述了"源图中的任意点 (x,y) 应该映射到目标图的哪个位置"。`warpPerspective` 遍历目标图的每个像素，通过逆矩阵反查它在源图中的位置，采样填充。

```python
src_pts = np.float32([[x1,y1],[x2,y2],[x3,y3],[x4,y4]])  # 原图中四角
dst_pts = np.float32([[0,0],[w,0],[w,h],[0,h]])           # 目标矩形
M = cv2.getPerspectiveTransform(src_pts, dst_pts)
warped = cv2.warpPerspective(img, M, (w, h))
```

### 真实案例
停车场车牌识别：摄像头以 30° 斜角拍摄，车牌在画面中呈梯形。先用透视变换将车牌区域矫正为正矩形，再送 OCR 识别——矫正后的识别准确率从 60% 提升到 95%。

## 概念 B：高斯滤波

### 一句话解释
高斯滤波用"中心重、边缘轻"的加权平均模糊图像——去噪声的同时比均值滤波更好地保留边缘。

### 生活类比
均值滤波像用一块干抹布均匀擦玻璃——边缘也一起模糊了。高斯滤波像用中间凸起的海绵擦——中间（像素本身）用力最大，越远的邻居影响越小。结果是噪点去掉了，物体的轮廓边界还在。sigma 越大 = 海绵越柔软、擦得越均匀模糊。

### 技术解释
高斯核的值由二维高斯函数计算：离中心越远，权重指数级衰减。核大小 `ksize` 必须是奇数。sigma=0 时 OpenCV 自动从 ksize 推算。核越大模糊越强但计算也越慢。

```python
blur = cv2.GaussianBlur(img, (5, 5), 0)  # 5×5 核，sigma 自动
blur_strong = cv2.GaussianBlur(img, (15, 15), 0)  # 更模糊
```

### 真实案例
工业相机拍摄金属零件时，金属表面的微小反光颗粒（噪点）会影响后续的边缘检测。`GaussianBlur(ksize=5)` 消除颗粒噪点，同时保留零件的外轮廓边缘。

## 概念 C：中值滤波与双边滤波

### 一句话解释
中值滤波取邻居的中位数——椒盐噪声的克星。双边滤波保留边缘的同时去噪——美颜相机的核心算法。

### 生活类比
中值滤波 = 班级里评选"中等身高"代表这个小组——极端值（200cm 或 140cm）直接被忽略。所以照片上的黑白椒盐噪点（极值）被完美去除。双边滤波 = 在"颜色相近"的邻居范围内做平均——边界处颜色跳变，所以边界两侧的像素互不干扰，边界被保留。

### 技术解释
`medianBlur` 对椒盐噪声效果远优于高斯。`bilateralFilter` 有两个 sigma：`sigmaColor`（颜色差异多大就视为边界）和 `sigmaSpace`（空间距离多远就不算邻居）。缺点是计算慢——比高斯慢 10-50 倍。

```python
median = cv2.medianBlur(img, 5)                    # ksize 必须是奇数
bilateral = cv2.bilateralFilter(img, 9, 75, 75)    # d, sigmaColor, sigmaSpace
```

### 真实案例
手机人像模式的美颜磨皮：用双边滤波去除皮肤上的斑点（噪点），同时保留眼睛、嘴唇、发际线等边界（边缘）。如果用了高斯滤波，整个脸都会糊掉。
