## 概念 A：VideoCapture — 视频的入口

### 一句话解释
`VideoCapture` 打开视频文件或摄像头，`.read()` 每次返回一帧——视频 = 帧序列 + 元数据（FPS、总帧数、分辨率）。

### 生活类比
电影放映机每秒闪过 24 格胶片。VideoCapture 就是慢速拆解这台放映机——打开片仓，一格一格地把胶片抽出来（read），记下当前的格数（frame index）和时刻（timestamp），还能知道总共多少格（frame count）和每秒几格（FPS）。

### 技术解释
```python
cap = cv2.VideoCapture("video.mp4")
fps = cap.get(cv2.CAP_PROP_FPS)           # 每秒帧数
total = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))  # 总帧数
w = int(cap.get(cv2.CAP_PROP_FRAME_WIDTH))
h = int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT))

while True:
    ret, frame = cap.read()  # ret=False 表示视频结束
    if not ret: break
    timestamp = frame_idx / fps  # 当前时间（秒）
    frame_idx += 1
cap.release()
```
关键点：(1) 必须调 `release()` 释放资源；(2) FPS 可能不准（可变帧率视频）——生产环境需要实际测量帧间隔；(3) `read()` 返回的 frame 是 BGR 格式。

### 真实案例
自动驾驶数据管道：车载摄像头以 30 FPS 录制。数据工程师用 `VideoCapture` 提取所有帧，按 10 Hz（每 3 帧取 1 帧）降采样，与激光雷达的 10 Hz 时间戳对齐。

## 概念 B：VideoWriter — 写出处理后的视频

### 一句话解释
`VideoWriter` 接收一帧帧图像，按指定 FPS 和编码格式合成新视频。

### 生活类比
VideoCapture 是拆胶片，VideoWriter 是装胶片——你把逐帧处理过的照片（比如每帧都画了检测框），按原来的速度重新装回放映机，生成新的 mp4。

### 技术解释
```python
fourcc = cv2.VideoWriter_fourcc(*'mp4v')  # MP4 编码
out = cv2.VideoWriter("output.mp4", fourcc, fps, (w, h))
out.write(processed_frame)   # 每帧写入
out.release()
```
常见 fourcc：`'mp4v'`（MP4）、`'XVID'`（AVI）、`'avc1'`（H.264，需额外安装）。写入帧的尺寸必须和初始化时声明的 (w, h) 完全一致，否则写入失败（静默）。

### 真实案例
监控系统：原始视频流入 → 每帧做背景减除 → 运动区域画红框 → VideoWriter 输出标注视频，供安保人员回看。

## 概念 C：背景减除与运动检测

### 一句话解释
背景减除器自动学习"背景长什么样"，然后用当前帧减去背景——剩下的就是运动物体（前景掩膜）。

### 生活类比
固定监控摄像头挂在天花板上。第一秒它看到空无一人的走廊（学习背景）。第 10 秒有人走过——"这个像素现在和背景不一样了！"→ 标记为前景。MOG2 算法还考虑了光照渐变（早晨→中午走廊变亮），旧背景像素值会慢慢更新。

### 技术解释
`createBackgroundSubtractorMOG2(history=500, varThreshold=16, detectShadows=True)` 基于高斯混合模型。`history` 控制"记住多久前的背景"（帧数），`varThreshold` 控制"偏差多大算前景"（越小越敏感）。检测到阴影时返回灰色(127)，通常用 `fg_mask == 255` 过滤掉。

```python
bg_sub = cv2.createBackgroundSubtractorMOG2(history=500, varThreshold=16)
fg_mask = bg_sub.apply(frame)  # 返回二值掩膜
# 形态学清理
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))
fg_mask = cv2.morphologyEx(fg_mask, cv2.MORPH_OPEN, kernel)
```

### 真实案例
智能交通灯：背景减除检测路口是否有等待车辆。连续 N 帧在 ROI 内检测到前景 → 触发绿灯。

## 概念 D：帧间追踪 — 从"检测运动"到"追踪轨迹"

### 一句话解释
连续帧中检测到同一物体后，关联"上一帧的物体 A"和"当前帧的物体 B"——它们是同一个东西——从而画出运动轨迹。

### 生活类比
你盯着一个足球比赛录像。第 5 帧球在 (100, 200)，第 6 帧球在 (105, 205)。你知道中间只隔了 1/30 秒，位置变化很小，所以这两帧里的白色圆球是同一个球。追踪逻辑就是：**"上一帧的 X 离这一帧的 Y 最近，它们大概率是同一物体"**。

### 技术解释
简单的帧间追踪 = 匈牙利算法（Hungarian Algorithm）做二分图匹配。对每帧前景掩膜找轮廓 → 计算每个轮廓的质心 → 用 `scipy.optimize.linear_sum_assignment` 找最近邻匹配。更稳健的方法：Kalman 滤波器预测下一帧位置，缩小搜索范围。

```python
# 简化版帧间追踪
prev_centroids = [(cx1,cy1), (cx2,cy2), ...]
curr_centroids = [(cx1,cy1), (cx2,cy2), ...]
# 计算距离矩阵
# 距离 < max_distance → 匹配成功 → 更新轨迹
# 未匹配的 curr → 新物体出现
# 未匹配的 prev → 物体消失
```

### 真实案例
体育赛事转播中，球员追踪系统实时追踪 22 个球员的位置，生成热力图和跑动距离统计——底层就是帧间轮廓匹配 + Kalman 滤波。

## 参考答案

Q1: B — `ret=False` 表示没有读到帧（到达视频末尾）

Q2: `p`, `4`, `v`; 指定视频的编码格式（压缩算法）

Q3: B — history 控制背景模型记住多少帧的像素历史，越大背景更新越慢

Q4: 正确 — 尺寸不匹配会导致 `write()` 静默失败

Q5: 交叉时两个质心距离很近，最近邻匹配可能"交换身份"——球 A 的轨迹跳到球 B 上。改进：用 Kalman 滤波预测每个物体的下一帧位置；或多加特征（颜色直方图）辅助匹配。
