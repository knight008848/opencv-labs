## 概念 A：ROI — 感兴趣区域

### 一句话解释
从整张图中切出你需要的局部区域，用 NumPy 切片语法 `img[y1:y2, x1:x2]`。

### 生活类比
你看手机时，注意力只集中在 6 寸屏幕上——你不会同时关注天花板和地板。ROI 就是告诉程序"别看全图，就看传送带上巴掌大的这块"。裁缝量尺寸只看领口那一小块布，不看整件衣服。

### 技术解释
ROI 是原图的一个**视图**（view），不是拷贝——修改 ROI 会直接影响原图。这是因为 NumPy 切片返回的是同一块内存的引用。需要独立副本时用 `roi.copy()`。切片语法 `img[行起点:行终点, 列起点:列终点]` 遵循 Python 左闭右开惯例。

```python
roi = img[50:200, 100:300]         # 切出 [50,200) 行、[100,300) 列
img[200:350, 100:300] = roi        # 把 ROI 粘贴到另一个位置
roi_copy = img[50:200, 100:300].copy()  # 独立副本，修改不影响原图
```

### 真实案例
自动驾驶中，摄像头画面的大部分是天空（无用的信息）。系统只提取路面区域（画面下半部分）做车道线检测——ROI 把无关区域排除在外，减少 60% 以上的计算量。

## 概念 B：直方图 — 像素值的统计分布

### 一句话解释
直方图数一数"图像中有多少个像素值为 0、多少个为 1……多少个为 255"，画成统计柱状图。

### 生活类比
全班 50 个学生的考试成绩——你用一个柱状图统计：0-59 分几人、60-69 分几人……90-100 分几人。直方图对图像做同样的事：0-50 亮度（暗）有多少像素、51-100（偏暗）有多少……200-255（亮）有多少。一张过曝的照片直方图会集中在右边（高亮区）。

### 技术解释
`cv2.calcHist([img], [0], None, [256], [0,256])` 计算通道 0（灰度或 B 通道）的直方图。返回 `(256, 1)` 的数组。可以通过直方图形状判断图像问题：集中在左侧=欠曝，集中在右侧=过曝，两端分布=低对比度。

```python
hist = cv2.calcHist([gray], [0], None, [256], [0, 256])
# hist[50] 表示亮度为 50 的像素有多少个
```

### 真实案例
监控摄像头的自动曝光控制：每秒计算当前画面的直方图，如果像素集中于暗区（夜晚模式），自动调高 ISO/曝光时间。工厂质检时，通过对比标准品和待检品的直方图相似度，快速发现颜色偏差。

## 概念 C：直方图均衡化

### 一句话解释
把像素值的分布"拉平"，让暗的地方变亮、亮的地方也不至于过曝，整体对比度大幅提升。

### 生活类比
一间教室里所有学生都挤在最后一排（像素全在暗区）。均衡化就像让老师安排重新排座位——学生均匀分布在整个教室。原来挤在一起的细节（暗区的纹理）一下子展开了，看得清清楚楚。

### 技术解释
`cv2.equalizeHist(gray)` 使用累积分布函数（CDF）将原像素值映射为新值，使输出直方图尽可能平坦。**只适用于灰度图**。彩色图需要在 HSV 空间中对 V 通道做均衡化（然后转回 BGR），否则颜色会失真。

```python
equ = cv2.equalizeHist(gray)  # 灰度图均衡化
# 彩色图正确做法：
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
hsv[:,:,2] = cv2.equalizeHist(hsv[:,:,2])  # 只调明度通道
result = cv2.cvtColor(hsv, cv2.COLOR_HSV2BGR)
```

### 真实案例
医疗 X 光片的原始图像对比度通常很低（骨骼和软组织灰度接近）。直方图均衡化后骨骼边缘清晰可见，帮助医生发现细微的骨裂。

## 参考答案

Q1: B — 行：60-10=50，列：80-20=60，即 50×60 像素

Q2: `roi.copy()` 或 `np.copy(roi)`

Q3: C — 欠曝 = 太暗 = 像素值都在低值区

Q4: 错误 — `equalizeHist` 只接受单通道灰度图

Q5: 在 BGR 空间分别均衡化会改变通道间的比例关系，导致颜色失真。HSV 空间将颜色（H）和亮度（V）解耦，只调 V 不影响色相。
