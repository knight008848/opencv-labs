## 概念 A：特征点与描述子

### 一句话解释
特征点 = 图片中"有辨识度"的位置（如角点）。描述子 = 把该位置周围的像素信息压缩成一个数字指纹，用于在不同图片中认出"这是同一个点"。

### 生活类比
你要在一个陌生城市找见面地点。特征点 = 地标建筑（东方明珠）——独一无二、容易找到、换个角度看还是能认出来。描述子 = 地标的"身份证"——"高 468 米、球形、位于浦东"。即使你在另一张城市照片里看到它（旋转了 90°、光线不同），身份证信息依然匹配。

### 技术解释
ORB（Oriented FAST + Rotated BRIEF）是免费、快速的特征检测器。FAST 找角点（周围一圈像素中连续 N 个都比中心亮/暗），BRIEF 用像素对的亮度比较生成 256-bit 二进制描述子（用汉明距离比较，极快）。

```python
orb = cv2.ORB_create(nfeatures=500, scaleFactor=1.2, nlevels=8)
keypoints, descriptors = orb.detectAndCompute(gray, None)
# keypoints[i].pt → (x, y) 坐标
# keypoints[i].size → 特征点尺度
# keypoints[i].angle → 特征点方向（ORB 有方向，所以旋转不变）
# descriptors.shape → (N, 32) 每个描述子 32 字节 = 256 bit
```

手机相机的全景模式：拍摄过程中，实时检测每帧的 ORB 特征点，匹配相邻帧的特征，计算变换矩阵，将新帧拼接到全景图中。

## 概念 B：特征匹配与 Ratio Test

### 一句话解释
对图 A 的每个特征点，在图 B 的描述子中找最相似的 K 个——如果最相似的和第二相似差距很大（ratio < 0.75），说明这个匹配"有把握"。

### 生活类比
你拿着一个人的照片在人群中找他。你找到了最像的两个人——第一个人很像（距离 = 10），第二个人有一点点像（距离 = 50）。ratio = 10/50 = 0.2 < 0.75 → 第一个人很可能就是你要找的。如果最像的和第二像的差距很小（ratio = 0.9），说明这个人长得太"大众脸"——你可能找错了。

### 技术解释
`BFMatcher` 暴力比对所有描述子，用汉明距离（XOR 后 bit 数）度量相似度。KNN 匹配找 K 个最近邻（通常 K=2）。Ratio Test：`match.distance / second_match.distance < 0.75` 才保留。0.75 是经验值——越小越严格（匹配越少但越准）。

```python
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=False)
matches = bf.knnMatch(des1, des2, k=2)
good = [m for m, n in matches if m.distance < 0.75 * n.distance]
```

## 概念 C：RANSAC 与单应矩阵

### 一句话解释
RANSAC 从一堆匹配中随机选最小样本（4 对点）拟合一个变换模型，然后统计"有多少其他点也符合这个模型"——反复试验，保留最多人同意的那个模型。

### 生活类比
两个吵架的人在调解——每个人各说各的版本（匹配对中的误匹配）。调解员（RANSAC）说："我随机选 4 个人的说法，假设他们说的是对的，然后看其他 96 个人谁支持这个版本。"试了 1000 次后，某次随机选到的 4 个人刚好都是说真话的——其他 80 个人也支持这个版本。"好，这个版本就是真相（内点），另外 20 个人在说谎（外点）。"

### 技术解释
`findHomography(src_pts, dst_pts, cv2.RANSAC, 5.0)` 计算从图 A 到图 B 的 3×3 单应矩阵。RANSAC 迭代次数自动计算。`5.0` 是重投影误差阈值（像素）——在此范围内的匹配视为内点。返回的单应矩阵 H 可以将图 A 的任意点映射到图 B：`dst = H @ src`。

```python
H, mask = cv2.findHomography(src_pts, dst_pts, cv2.RANSAC, 5.0)
inlier_ratio = mask.sum() / len(mask)  # 内点比例，越高越可靠
```

## 参考答案

Q1: C — 纯色区域没有纹理变化，无法产生稳定的特征点

Q2: Oriented FAST; Rotated BRIEF; 汉明（Hamming）

Q3: B — 最近距离与次近距离的比值 < 0.75

Q4: 错误 — RANSAC 是随机算法，不保证全局最优，但通常能找到足够好的解

Q5: ORB 免费开源且速度快（二进制描述子+汉明距离），SIFT 有专利限制且计算量大。ORB 适合实时应用和入门学习，SIFT 精度略高但工程代价大。
