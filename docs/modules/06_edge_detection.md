## 概念 A：Sobel 算子 — 计算亮度变化的梯度

### 一句话解释
Sobel 算子计算每个像素在 X 方向和 Y 方向的亮度变化率——变化越剧烈的地方越可能是边缘。

### 生活类比
你闭着眼睛用手指摸桌面，手指缓慢移动。当手指从桌面滑到桌边（突然悬空），你会立刻感觉到"这里断了"——这就是梯度。Sobel 把手指换成数学公式：左边像素减右边像素（X 方向），上边像素减下边像素（Y 方向），变化大的地方就是边缘。

### 技术解释
Sobel 核是 3×3 的矩阵。X 方向核 `[[-1,0,1],[-2,0,2],[-1,0,1]]` 检测垂直边缘（亮度在水平方向的变化）。Y 方向核 `[[-1,-2,-1],[0,0,0],[1,2,1]]` 检测水平边缘。总梯度 `G = sqrt(Gx² + Gy²)` 或近似 `|Gx| + |Gy|`。

```python
sobel_x = cv2.Sobel(gray, cv2.CV_64F, 1, 0, ksize=3)  # X 方向梯度
sobel_y = cv2.Sobel(gray, cv2.CV_64F, 0, 1, ksize=3)  # Y 方向梯度
sobel_combined = cv2.magnitude(sobel_x, sobel_y)        # 合并
```

### 真实案例
PCB 电路板缺陷检测：正常走线边缘是笔直的，Sobel 检测到的边缘也是直的。如果某处边缘弯曲或断开，就是短路或断路的信号。

## 概念 B：Canny 边缘检测

### 一句话解释
Canny 是一个多阶段边缘检测器：先降噪，再找梯度，然后只保留梯度最大的（细线），最后用双阈值筛选——高于高阈值的肯定是边缘，低于低阈值的肯定不是，中间的看它是否连着确定边缘。

### 生活类比
你在一片嘈杂的广场上找人——第一步排除干扰（降噪），第二步听到声音的方向（梯度），第三步只锁定声音最大的来源（非极大值抑制），第四步："这个方向声音超过 80 分贝的肯定是人（高阈值），低于 30 分贝的肯定是风噪（低阈值），50 分贝的——如果你旁边有个已经确认是人的声源，那这个也可能是人（滞后阈值）。"

### 技术解释
Canny 的两个阈值（`threshold1` 低, `threshold2` 高）是最关键参数。通常 `t2 ≈ 2×t1`。阈值越低越敏感（边缘多+噪点多），阈值越高越干净（边缘少+可能漏检）。输入必须是灰度图，且建议先用高斯模糊降噪。

```python
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
blurred = cv2.GaussianBlur(gray, (5, 5), 0)    # 先降噪
edges = cv2.Canny(blurred, 50, 150)            # 低阈值=50, 高阈值=150
```

### 真实案例
自动驾驶车道线检测：前方画面经过 Canny 后，车道线呈现为两条清晰的白色边缘。Hough 变换在此基础上检测直线，确定车道位置。

## 概念 C：Laplacian 与边缘类型对比

### 一句话解释
Laplacian 是二阶导数——不关心亮度是变大还是变小，只关心变化率本身的"变化"。对噪点极其敏感，通常不直接使用。

### 生活类比
Sobel 告诉你"速度"（亮度变化快慢），Laplacian 告诉你"加速度"（变化本身在加速还是减速）。加速度在拐点处最大——恰好就是边缘的正中心。但任何微小的抖动（噪点）也有加速度，所以 Laplacian 满图都是噪点。

### 技术解释
Laplacian 核 `[[0,1,0],[1,-4,1],[0,1,0]]` 计算二阶导数。通常先高斯模糊再用 Laplacian（合起来叫 LoG = Laplacian of Gaussian），效果才可控。实际工程中 Canny 使用频率远高于 Laplacian。

```python
laplacian = cv2.Laplacian(gray, cv2.CV_64F)
# 几乎总是需要先 GaussianBlur！
```

### 真实案例
虽然 Laplacian 不常用于边缘检测，但它在**图像锐化**中很有用：原图加上 Laplacian（负值）可以增强边缘的对比度，让照片看起来更"锐"。
