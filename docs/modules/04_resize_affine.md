## 概念 A：图像缩放

### 一句话解释
`resize` 改变图像的像素尺寸——可以用绝对尺寸 `(宽, 高)` 或相对比例 `(fx, fy)`。

### 生活类比
缩放 = 复印机上的放大/缩小按钮。缩小到 50%：4 个像素合并成 1 个（取平均），画面变小但内容全在。放大到 200%：1 个像素拆成 4 个（需要"猜"中间的颜色——插值）。`INTER_AREA` 像"聪明地平均"（缩小用），`INTER_CUBIC` 像"看周围 16 个邻居来猜"（放大用）。

### 技术解释
```python
small = cv2.resize(img, None, fx=0.5, fy=0.5, interpolation=cv2.INTER_AREA)
large = cv2.resize(img, (500, 300), interpolation=cv2.INTER_CUBIC)  # 注意：(宽,高)
```
关键点：(1) 直接指定尺寸时顺序是 `(宽, 高)`，与 shape 的 `(高, 宽)` 相反；(2) 不保持纵横比会导致图像被拉伸变形；(3) `INTER_AREA` 是缩小的最佳选择（无摩尔纹），`INTER_CUBIC` 是放大的较佳选择。

### 真实案例
深度学习模型（如 YOLO）要求固定输入尺寸 640×640。原始照片可能是 4000×3000——需要先缩放到 640×640，短边用灰度填充保持比例。这一步的插值质量直接影响检测精度。

## 概念 B：仿射变换 — 旋转、平移、缩放

### 一句话解释
仿射变换是平面内"保持平行线依然平行"的任何操作——旋转、平移、缩放、错切。

### 生活类比
一张纸放在桌上，你可以旋转它（rotate）、推它（translate）、拉伸它（scale）。纸永远是平的，上面画的格子线永远是直的，平行线永远不会相交。但你不能把纸折弯——那是透视变换的事。

### 技术解释
仿射变换用 2×3 矩阵表示，由 3 对对应点确定。实际常用 `getRotationMatrix2D(center, angle, scale)` 直接获得旋转矩阵，再用 `warpAffine` 执行。旋转后图像角落可能被裁切——需要计算新的画布尺寸来避免。

```python
h, w = img.shape[:2]
M = cv2.getRotationMatrix2D((w/2, h/2), 45, 1.0)  # 绕中心旋转45°
rotated = cv2.warpAffine(img, M, (w, h))
```

### 真实案例
人脸对齐：检测到人脸关键点（眼睛、鼻子、嘴）后，用仿射变换将歪头的人脸旋转摆正，确保所有人脸输入到识别模型时都是"端正"的。

## 概念 C：图像金字塔

### 一句话解释
图像金字塔是一系列逐级缩小（或放大）的图像，每一级都是上一级的一半大小——像一座分辨率递减的金字塔。

### 生活类比
Google 地图的缩放层级——最底层是街道细节（原始分辨率），上一层是街区，再上是城市、省份、国家。图像金字塔让你可以"先粗略定位，再精确分析"。

### 技术解释
高斯金字塔：每次 `pyrDown` 将尺寸减半——先用高斯核模糊（去高频噪声防止混叠），再删掉偶数列和偶数行。拉普拉斯金字塔：存储每层 `pyrUp(pyrDown(img)) - img` 的差值——即模糊丢失的"细节层"。

```python
smaller = cv2.pyrDown(img)   # 尺寸减半
bigger = cv2.pyrUp(img)      # 尺寸加倍（不能恢复丢失的细节）
```

### 真实案例
人脸检测中的滑窗策略：不是每一帧都全分辨率扫描，而是先生成金字塔，在低分辨率层快速筛选候选区域，只有候选区才在高分辨率层精细确认。

## 参考答案

Q1: C — `INTER_AREA` 是缩小的最佳选择，能避免摩尔纹

Q2: 300; 200; (宽, 高)

Q3: A — 仿射变换 = 线性变换（旋转/缩放/错切）+ 平移，平移是仿射的一个特例

Q4: 正确 — 正方形旋转 45° 后对角线 = 边长×√2，比原来宽

Q5: 不模糊的话直接删行列会产生**混叠效应**（aliasing）——高频细节（如细条纹）会变成奇怪的锯齿或波纹。高斯模糊先去除高频分量。
