## 概念 A：图像即矩阵

### 一句话解释
计算机把图片存为「高 × 宽 × 通道数」的三维数字表格，每个格子是一个 0-255 的整数。

### 生活类比
想象一面由彩色马赛克瓷砖拼成的壁画。每块瓷砖代表一个像素，颜色用三个数字记录（蓝多少、绿多少、红多少）。整面墙 = 一个巨大的 Excel 表格，行数 = 图片高度，列数 = 图片宽度，每个格子里填了 3 个数字。

### 技术解释
OpenCV 用 NumPy 的 `ndarray` 存储图像。一张 480×640 的彩色图在内存中是 `shape=(480, 640, 3)` 的数组，`dtype=uint8`。访问像素用 `img[y, x]`（注意：先行后列，即先 y 后 x）。三通道按 BGR 顺序排列（不是 RGB！）。

```python
import cv2
img = cv2.imread("photo.jpg")       # 读取 → ndarray
print(img.shape)                     # (480, 640, 3)
print(img.dtype)                     # uint8
b, g, r = img[100, 200]             # 访问第 100 行、第 200 列的像素
```

### 真实案例
工业相机的原始输出就是一帧帧的图像矩阵。质检系统读取矩阵后，检查特定区域的像素值是否在合格范围内——例如，药片颜色偏暗说明受潮。

## 概念 B：BGR → RGB → 灰度

### 一句话解释
OpenCV 默认用 BGR（蓝绿红）存颜色；人类习惯 RGB（红绿蓝）；灰度图只有一个通道，丢弃颜色只保留亮度。

### 生活类比
BGR 和 RGB 就像两种不同的调料瓶排列——同样的三种调料（蓝、绿、红），瓶子的顺序不同。你用 Matplotlib 做菜时如果拿错了瓶子（把 BGR 当 RGB 用），炒出来的菜颜色就偏蓝。灰度图则是把所有调料混在一起，只看"咸淡"（亮度），不管具体什么颜色。

### 技术解释
`cv2.cvtColor(img, cv2.COLOR_BGR2RGB)` 把第 0 和第 2 通道交换。`cv2.COLOR_BGR2GRAY` 用公式 `Gray = 0.114×B + 0.587×G + 0.299×R` 加权合并——绿色权重最大，因为人眼对绿色最敏感。

```python
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)  # (480, 640) 单通道
rgb = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)     # (480, 640, 3) 通道顺序变了
```

### 真实案例
人脸检测算法通常在灰度图上运行——丢弃颜色信息后计算量减少到 1/3，而人脸轮廓（亮度变化）依然保留。几乎所有目标检测的第一步都是转灰度。

## 概念 C：`imshow` 与窗口控制

### 一句话解释
`imshow` 弹出一个窗口显示图像；`waitKey` 控制窗口停留多久；`destroyAllWindows` 关闭所有窗口。

### 生活类比
`imshow` 就像把照片贴在墙上给人看。`waitKey(0)` 等于"等人看完再把照片取下来"（按任意键关闭）。`waitKey(1000)` 等于"只贴 1 秒，自动取下"。如果不调 `waitKey`，照片一闪而过根本看不清。

### 技术解释
`waitKey(n)` 等待 n 毫秒。n=0 表示无限等待直到按键。返回值是按键的 ASCII 码（如按 ESC=27）。这是 OpenCV 的事件循环——没有它窗口不会刷新。

```python
cv2.imshow("My Image", img)
key = cv2.waitKey(0)          # 等待按键
if key == 27:                 # ESC
    cv2.destroyAllWindows()
```

### 真实案例
调试视觉算法时，在每一步处理后 `imshow` 中间结果——原图 → 灰度 → 模糊 → 边缘→ 轮廓，每一步都能看到效果，方便定位 bug。
