## 最终项目：具身视觉数据管道

### 项目描述

输入一段你用手机拍摄的桌面操作视频（30-60 秒 MP4），程序自动完成：

```
MP4视频 
  → 帧提取 + 时间戳对齐
  → 多 ROI 分窗（全局/局部/入口 → 模拟多相机）
  → 每个 ROI 独立做：颜色过滤 + 边缘检测 + 轮廓分析
  → 帧间物体追踪（同一物体跨帧位置变化）
  → 打包为结构化数据（JSON + NPZ）
```

### 为什么这个项目贴近具身智能

具身数据管道的核心问题是：**如何从视频流中提取时空对齐的观察向量，供 VLA 模型使用**。这个项目复现了：
- 多视角观察（多 ROI = 多相机模拟）
- 时空特征对齐（同一物体在连续帧中的特征变化）
- 结构化输出（JSON 描述语义，NPZ 存储特征向量）

不需要多台相机、不需要机械臂、不需要外部数据集——**一部手机拍 30 秒视频即可**。

### 用到的技能（对应 domain_map.md 概念）

| 步骤 | 技能 | 概念编号 |
|:---|:---|:---|
| 视频读取 | `VideoCapture`、FPS、帧索引 | #20 |
| 帧预处理 | `cvtColor`、`GaussianBlur` | #5, #10 |
| ROI 分窗 | NumPy 切片 | #7 |
| 颜色过滤 | HSV + `inRange` | #5 |
| 边缘检测 | `Canny` | #11 |
| 轮廓检测 | `findContours`、`contourArea`、`boundingRect` | #12 |
| 帧间追踪 | 轮廓质心匹配、帧间距离 | 综合运用 |
| 数据打包 | JSON + `np.savez` | 工程实践 |

### 验收标准
- [ ] `python experiments/day_30_demo.py data/raw/my_video.mp4` 一行命令跑通
- [ ] 自动从 MP4 中提取帧（可配置帧间隔）
- [ ] 至少 3 个 ROI 窗口各自完成特征提取
- [ ] 帧间物体追踪轨迹可视化（在帧上画运动轨迹线）
- [ ] 输出包含：逐帧 JSON（物体位置/颜色/形状）+ NPZ（特征向量）+ 可视化视频
- [ ] 可录制一段完整的演示视频（屏幕录制，展示从命令行到结果的全过程）
